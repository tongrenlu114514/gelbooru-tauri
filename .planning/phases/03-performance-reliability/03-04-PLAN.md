<objective>
Add global HTTP rate limiting to the HttpClient in http.rs. Enforce a 500ms gap between consecutive HTTP requests globally (covering all Gelbooru search and image download requests) using `RwLock<Instant>` to track the last request timestamp. This respects Gelbooru's implicit rate limits and ensures responsible API usage.
</objective>

<context>
@src-tauri/src/services/http.rs
@src-tauri/src/commands/gelbooru.rs
@src-tauri/src/commands/download.rs

The current `HttpClient` (lines 1-141 in http.rs) has no rate limiting. Every call to `get()`, `get_image_with_referer()`, or `download_image()` proceeds immediately.

The fix: add a `last_request_time: RwLock<Instant>` field to `HttpClient`. Before every HTTP call, compute the elapsed time since `last_request_time`, and `tokio::time::sleep` for the remaining gap if less than 500ms has elapsed. Then update `last_request_time` to `Instant::now()`.

Key context:
- `HttpClient` is used via `HTTP_CLIENT.read().await` from gelbooru.rs and download.rs
- `HttpClient::new()` constructs the client — initialize `last_request_time` there
- `HttpClient::set_proxy()` rebuilds the client — update `last_request_time` in the new client
- All three public HTTP methods (`get`, `get_image_with_referer`, `download_image`) must call the rate limit enforcement
- Use `tokio::time::sleep` (NOT `std::thread::sleep`) since this is async code
</context>

   ```rust
   /// Enforce global rate limit: ensure at least 500ms between consecutive requests.
   async fn wait_for_rate_limit(&self) {
       let min_gap = Duration::from_millis(RATE_LIMIT_INTERVAL_MS);
       let elapsed = {
           let last = self.last_request_time.read().await;
           last.elapsed()
       };
       if elapsed < min_gap {
           sleep(min_gap - elapsed).await;
       }
       *self.last_request_time.write().await = Instant::now();
   }
   ```

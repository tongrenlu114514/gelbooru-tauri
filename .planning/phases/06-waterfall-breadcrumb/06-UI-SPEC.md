## Component Inventory

### Breadcrumb Bar

Replaces `.path-bar` div. Sits above `.folder-list`.

```
<div class="breadcrumb-bar">
  <n-breadcrumb>
    <n-breadcrumb-item
      v-for="(segment, i) in breadcrumbSegments"
      :key="i"
      :clickable="i < breadcrumbSegments.length - 1"
    >
      <n-icon :size="14" style="margin-right: 4px">
        <FolderOpenOutline />
      </n-icon>
      {{ segment }}
    </n-breadcrumb-item>
  </n-breadcrumb>
</div>
```

**Visual contract:**
- Container: no background, no border, `margin-bottom: 8px`, left-aligned
- Separator: naive-ui default `>` character
- Folder icon: `#f0a020` (accent color)
- Text: `13px`, `#333`, not bold
- Current (last) segment: non-clickable, slightly bolder (`500`)
- Ancestor segments: `#007aff` (blue, naive-ui default link color), cursor pointer
- Hover: naive-ui default underline

**Interactive behavior:**
- Click ancestor segment → navigate to that folder + scroll to first image in viewport
- If target === current folder: do nothing (no scroll)
- If target folder is not in viewport: scroll after navigation

### Folder List

**Already exists.** Phase 06 keeps `.folder-list` horizontal pills above masonry grid unchanged.

**Visual contract:**
- Container: `margin-bottom: 8px`, `min-height: 32px`
- Items: `.folder-item` with `gap: 4px`, `padding: 4px 8px`, `border-radius: 4px`
- Icon: `<n-icon :size="14" color="#f0a020">` (accent)
- Name: `13px`, `#333`
- Count badge: `11px`, `#999`, `background: #f5f5f5`, `padding: 1px 5px`, `border-radius: 8px`
- Active state: `background: #e8f0fe`
- Hover: `background: #f0f0f0`
- Divider: `|` character, `#ddd`, `12px`

### Masonry Grid

Replaces `.content-grid` (CSS Grid `auto-fill, minmax(160px, 1fr)`) with `MasonryWall`.

```
<MasonryWall
  :items="displayItems"
  :column-width="160"
  :gap="4"
  :min-columns="1"
  class="content-grid"
>
  <template #default="{ item, index }">
    <div
      class="gallery-card"
      :class="{ 'folder-card': item._type === 'folder' }"
      :data-image-path="item._type === 'image' ? item.path : (item.thumbnail ?? '')"
    />
  </template>
</MasonryWall>
```

**Visual contract:**
- Grid container: `padding: 4px`, `gap: 4px` (both from token `--space-xs`)
- Card: `aspect-ratio: 1`, `border-radius: 4px`, `background: #fff`, no border
- Card hover: `box-shadow: 0 2px 8px rgba(0,0,0,0.15)`, `z-index: 10`
- Card hover overlay: `::after` gradient, `opacity: 0 → 1`
- Card filename: `12px`, white, `bottom: 8px; left: 8px; right: 8px`, ellipsis
- Skeleton: identical to existing (12-skeleton grid, `160px` height, `border-radius: 4px`)
- Empty state: unchanged from Phase 05

**Transition behavior:**
- When folder navigation occurs: skeleton shows while loading, then MasonryWall fades in
- Scroll-to-first-card: `behavior: 'smooth'`, `block: 'start'`
- Viewport check before scroll: skip scroll if first card already in view (`rect.top >= 0 && rect.bottom <= window.innerHeight`)

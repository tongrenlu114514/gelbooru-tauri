## 当前实现

```css
.gallery-cards {
  columns: 160px 1;   /* 160px最小列宽，尽量少列 */
  column-gap: 4px;
}
```

`columns: 160px 1` 语义：最小列宽160px，列数从1开始增长。问题：当容器宽度为600px时只有4列；当容器宽度为1200px时只有7列——列数随容器变化，但用户无法控制。

## 目标实现

```css
.gallery-cards {
  columns: 160px 3;   /* 固定160px列宽，最多3列 */
  column-gap: 4px;
}
```

语义：最小160px列宽，最多3列，超出则溢出（可横向滚动）或列宽自动扩大到填满容器。

**方案选择：**

`columns: 160px N` 中N为最大列数，列宽固定160px，列数随容器宽度在1~N之间自动调整。当N=3，容器1200px时恰好填满3列x160px=480px（加间距），视觉效果好。

注意：如果容器比N列更宽，列宽会扩展超过160px。如果需要严格固定160px宽度，应改用CSS Grid + `auto-fill` / `auto-fit`，但CSS columns天然支持Masonry的break-inside:avoid，无需JavaScript计算高度，综合来看`columns: 160px 3`是平衡点。
</context>

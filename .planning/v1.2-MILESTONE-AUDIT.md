## Critical Gaps (Blockers)

### Gap 1: IndexingService never initialized (CRITICAL — IDX-02, IDX-04)

**Root Cause:** `setup_indexing_service()` defined at `indexing.rs:531` but never called in `main.rs`.

**What happens at runtime:**
- `app_handle.try_state::<IndexingService>()` always returns `None`
- Background mpsc channel never created
- Thumbnails never written to `app_data_dir/thumbnails/`
- IDX-02 (thumbnails in cache) and IDX-04 (background generation) are **completely non-functional**

**Fix (2 lines in main.rs):**
```rust
// After: .manage(DbState(Mutex::new(database.clone())))
use commands::indexing::setup_indexing_service;
setup_indexing_service(&app_handle, Arc::new(database.clone()));
```

### Gap 2: 5 Tauri commands not registered in invoke_handler (HIGH — IDX-01, IDX-03)

**Root Cause:** `commands::indexing::*` never added to `generate_handler![...]` in `main.rs`.

**Impact:**
- `scan_gallery` — cannot index gallery images from UI
- `get_indexed_images` — cannot query indexed images
- `generate_thumbnail` — cannot generate thumbnails on demand
- `get_thumbnail_path` — cannot retrieve thumbnail paths
- `start_background_thumbnail_scan` — cannot trigger background scan

**Fix:** Add to `generate_handler![...]` in main.rs:
```rust
commands::indexing::scan_gallery,
commands::indexing::get_indexed_images,
commands::indexing::generate_thumbnail,
commands::indexing::get_thumbnail_path,
commands::indexing::start_background_thumbnail_scan,
```
